Hexo 使用入门

发表于2025-12-07|更新于2025-12-09|技术/Hexo
|浏览量:

Hexo 是什么？

Hexo 是一款基于 Node.js 的静态博客框架，可以把我们用 Markdown 等标记语言写好的文章，快速转换成静态的 HTML 页面。
它最大的特点就是 完全不依赖数据库和后台服务，生成的都是可以直接丢到服务器上的静态文件。

简单理解就是：

你负责写文章，Hexo 负责帮你“打包”成一个完整的网站。

Hexo 适合哪些人？

先说结论：Hexo 不是“纯小白向工具”。

它更适合下面这类人：

  • 懂一点点命令行操作

  • 遇到报错愿意查文档、搜问题

  • 有一点前端基础，或者愿意慢慢折腾

  • 想搭一个属于自己的技术博客/知识库的人

虽然 Hexo 有很多现成的开源主题可以直接用，但如果你想“魔改”页面样式，比如改布局、动画、配色，那基本就绕不开 HTML / CSS / JavaScript 这些基础。

一句话总结：

Hexo 很友好，但它更喜欢和“爱折腾的人”做朋友。

Hexo 有哪些优点？

Hexo 的优势非常明显，尤其适合个人博客场景：

  • 纯静态页面：访问速度快，而且安全性更高

  • 成本极低：可以直接部署到 GitHub Pages 上，几乎是“零成本上线”

  • 高度可定制：主题、插件都很丰富，爱怎么折腾就怎么折腾

  • 主题生态成熟：有大量现成主题可以直接拿来用

  • 天然适配 Git：文章就是文件，非常适合用 Git 做版本管理

核心优点一句话总结：

轻、快、便宜、可玩性强。

整个过程不用数据库、不用后端服务，非常适合做个人博客、文档站之类的场景。

2）部署到 GitHub Pages

这是最常见的方式：

  • 代码托管到 GitHub

  • 使用 GitHub Pages 提供的免费静态托管

  • 一个域名就能公网访问

特点是：省钱、省心、适合个人使用。

3）部署到自己的服务器（Nginx + Linux）

如果你有自己的云服务器：

  • 用 Hexo 生成静态文件

  • 用 Nginx 做 Web 服务

  • 直接对外提供访问

这种方式：

  • 更灵活

  • 更可控

  • 适合有一定运维基础的人

4）自动化部署（GitHub Actions）

进阶玩法：

  • 每次 git push

  • 自动触发 CI 流水线

  • 自动构建 + 自动部署

一句话总结就是：

写完文章一提交，网站自动更新，几乎不用管运维。
